### 1. Главная страница (`/main/home`)
**Файл:** `lib/presentation/screens/main_menu/home_screen.dart`

#### 1.1 AppBar (верхняя панель)
- **1.1.1** User видит заголовок "Лежандр"
- **1.1.2** User кликает иконку уведомлений (колокольчик) — заглушка, функционал в разработке

#### 1.2 Карточка приветствия (`_GreetingCard`)
- **1.2.1** User видит приветствие в зависимости от времени суток:
  - "Доброе утро" (до 12:00)
  - "Добрый день" (12:00-18:00)
  - "Доброе вечер" (после 18:00)
- **1.2.2** User видит своё имя пользователя
- **1.2.3** User видит текущий стрик (количество дней решения задач без пропусков)
  - Иконка 🔥 оранжевая при стрике > 0, серая при стрике = 0
- **1.2.4** User видит количество решённых сегодня задач
- **1.2.5** User видит иконку приложения (lezhandr_icon.webp) справа в карточке

#### 1.3 Карточка мотивации (`MotivationCard`)
- **1.3.1** User видит мотивационную фразу на основе:
  - Текущего стрика
  - Количества решённых сегодня задач
  - Общего XP
- **1.3.2** User видит автора цитаты (если есть)

#### 1.4 Карточка быстрой статистики (`_QuickStatsCard`)
- **1.4.1** User видит общий заработанный XP (иконка ⭐)
- **1.4.2** User видит количество сердец (текущее/максимальное)

#### 1.5 Карточка активных задач (`_ActiveSolutionsCard`)

**При отсутствии активных задач:**
- **1.5.1** User видит иконку пустого списка и текст "Нет активных задач"
- **1.5.2** User кликает "Новая задача" → переход в Библиотеку

**При наличии активных задач:**
- **1.5.3** User видит до 3 активных задач с названием и временем решения
- **1.5.4** User видит бейдж с количеством активных задач
- **1.5.5** User кликает ▶ у задачи → переход в сессию решения
- **1.5.6** User кликает "Новая задача" → переход в Библиотеку

#### 1.6 Быстрые действия
- **1.6.1** User кликает "Начать решение" → переход в Библиотеку
- **1.6.2** User кликает "Анализ концепций" → переход на страницу концептов

#### 1.7 FAB (кнопка "Решать")
- **1.7.1** User кликает FAB "Решать" → переход в Библиотеку

#### 1.8 Pull-to-refresh
- **1.8.1** User тянет вниз → обновление данных (активные задачи, геймификация, пользователь)

### 3. Библиотека задач (`/main/library`)
**Файл:** `lib/presentation/screens/library/library_screen.dart`

#### 3.1 AppBar
- **3.1.1** User видит заголовок "Библиотека"
- **3.1.2** User кликает иконку ? → открывается help bottom sheet с описанием возможностей
- **3.1.3** User кликает иконку поиска → открывается диалог поиска

#### 3.2 Фильтр по источникам
- **3.2.1** User видит горизонтальный список источников (chips)
- **3.2.2** User кликает "Все" → показываются задачи из всех источников
- **3.2.3** User кликает конкретный источник → фильтрация по источнику
- **3.2.4** Источники на модерации показываются с бейджем "на модерации" (оранжевый)
- **3.2.5** Отклонённые источники показываются с бейджем "отклонено" (красный)

#### 3.3 Список задач
- **3.3.1** User видит счётчик "Всего задач: N"
- **3.3.2** User видит карточки задач с:
  - Номером/названием
  - Источником
  - Именем автора (кто добавил)
  - Превью условия (текст или "Есть фото условия")
  - Тегами
  - Бейджем "Активно" для задач в процессе решения
  - Бейджем модерации (если задача pending/rejected)
- **3.3.3** User кликает карточку задачи → переход на страницу задачи
- **3.3.4** User скроллит вниз → загрузка следующих 20 задач (infinite scroll)

#### 3.4 Поиск задач
- **3.4.1** User открывает диалог поиска
- **3.4.2** User вводит текст → фильтрация по тексту
- **3.4.3** User кликает "Сбросить" → очистка поиска
- **3.4.4** User кликает "Найти" → применение фильтра

#### 3.5 Создание новой задачи (FAB)
- **3.5.1** User кликает "Новая задача" → открывается bottom sheet создания
- **3.5.2** User выбирает источник из списка или создаёт новый (inline)
- **3.5.3** User вводит номер/название задачи (обязательно)
- **3.5.4** User добавляет теги (с автоподсказками)
- **3.5.5** User вводит текст условия (опционально, поддержка LaTeX)
- **3.5.6** User свайпит вниз → закрытие bottom sheet
- **3.5.7** User кликает "Создать" → задача создаётся
- **3.5.8** После создания User видит диалог "Задача создана! Добавить фото?"
  - "Позже" → переход на страницу задачи
  - "Добавить фото" → переход на экран камеры

### 4. Страница задачи (`/problems/{id}`)
**Файл:** `lib/presentation/screens/problems/problem_detail_screen.dart`

#### 4.1 AppBar
- **4.1.1** User кликает иконку ? → открывается help bottom sheet с описанием возможностей
- **4.1.2** User кликает ⋮ → открывается меню действий с условием

#### 4.2 Header (источник + автор)
**Файл:** `lib/presentation/screens/problems/widgets/problem_header.dart`
- **4.2.1** User видит источник задачи (бейдж)
- **4.2.2** User видит номер/название задачи
- **4.2.3** User видит имя автора задачи

#### 4.3 Теги
**Файл:** `lib/presentation/screens/problems/widgets/tags_editor.dart`
- **4.3.1** User видит теги задачи (chips)
- **4.3.2** Если User — автор и задача на модерации, видит кнопку "Теги" для редактирования
- **4.3.3** User кликает "Теги" → bottom sheet редактирования с поиском/созданием тегов
- **4.3.4** Теги на модерации показываются с бейджем (оранжевый/красный)

#### 4.4 Карточка условия
**Файл:** `lib/presentation/screens/problems/widgets/condition_card.dart`
- **4.4.1** User видит текст условия (если есть) с поддержкой LaTeX
- **4.4.2** User видит фото условия (если есть)
- **4.4.3** Если есть и текст, и фото — User кликает "Фото/Текст" для переключения
- **4.4.4** Кнопки редактирования (только для автора задачи на модерации):
  - Иконка ✏️ → редактирование текста
  - Иконка ✨ → OCR (если есть фото)
- **4.4.5** Для approved/rejected задач кнопки редактирования неактивны (disabled)
- **4.4.6** Автор кликает ⋮ → меню действий:
  - "Редактировать текст" (только для pending)
  - "Загрузить фото" (только для pending)
  - "Распознать текст (OCR)" — если есть фото
  - "Открыть фото" — просмотр в полном размере
- **4.4.7** Для approved задач показывается сообщение "Задача прошла модерацию"
- **4.4.8** Для rejected задач показывается сообщение "Задача отклонена"

#### 4.5 Редактирование текста условия (bottom sheet)
- **4.5.1** User видит заголовок "Редактировать условие"
- **4.5.2** User видит TextField с текущим текстом (6 строк, поддержка LaTeX)
- **4.5.3** User кликает "Отмена" → закрытие без сохранения
- **4.5.4** User кликает "Сохранить" → сохранение и закрытие
- **4.5.5** User свайпит вниз → закрытие без сохранения
- **4.5.6** При открытой клавиатуре контент скроллится

#### 4.6 Карточка концептов
**Файл:** `lib/presentation/screens/problems/widgets/problem_concepts_section.dart`
- **4.6.1** User видит заголовок "Концепты" и кнопку "Анализ"
- **4.6.2** При отсутствии концептов — сообщение с предложением анализа
- **4.6.3** User кликает "Анализ" → выбор AI-персоны → запуск анализа
- **4.6.4** При наличии концептов User видит список с:
  - Названием концепта
  - Процентом релевантности
  - Цветом по релевантности (зелёный ≥80%, оранжевый ≥50%, серый <50%)
- **4.6.5** User кликает концепт → раскрывается детализация:
  - "Объяснение" — почему концепт релевантен
  - "Описание концепта" — общее описание
  - "Практическое применение" — как использовать
- **4.6.6** User кликает развёрнутый концепт → сворачивание

#### 4.7 Список решений
**Файл:** `lib/presentation/screens/problems/widgets/solutions_list.dart`
- **4.7.1** User видит список решений с:
  - Статусом (активно/завершено/приостановлено)
  - Именем решателя
  - Временем решения
  - Заработанным XP
  - Иконками наличия текста/фото
  - Бейджем модерации (если решение pending/rejected)
- **4.7.2** Если решение активно и User — не автор → иконка 🔒
- **4.7.3** User кликает заблокированное решение → сообщение "Другой пользователь еще решает"
- **4.7.4** User кликает своё активное решение → переход в сессию
- **4.7.5** User кликает завершённое решение → переход на страницу решения

#### 4.8 FAB "Решать"
- **4.8.1** User кликает "Решать" → создание нового решения → переход в сессию

### 5. Сессия решения (`/session/{id}`)
**Файл:** `lib/presentation/screens/solutions/solution_session_screen.dart`

#### 5.1 Первый запуск (intro)
- **5.1.1** При первом входе в сессию показывается bottom sheet с инструкцией:
  - "Время подумать"
  - Совет решать самостоятельно
  - Возможности: вопросы, озарения, подсказки
  - Предупреждение "Не спешите!"
- **5.1.2** User кликает "Понятно, начинаю!" → закрытие intro

#### 5.2 Таймер
- **5.2.1** User видит время сессии в минутах (или часах:минутах если >1 часа)
- **5.2.2** User видит "Ранее: X.X мин" если это продолжение сессии

#### 5.3 Карточка условия задачи
- **5.3.1** User видит превью условия (текст или фото)
- **5.3.2** User кликает на превью → переход на страницу задачи

#### 5.4 Мотивация
- **5.4.1** User видит мотивационную карточку (меняется раз в минуту)

#### 5.5 Озарения (Epiphanies)
- **5.5.1** User видит секцию "Озарения" с кнопкой +
- **5.5.2** User кликает + → bottom sheet для добавления озарения:
  - Текст озарения (обязательно)
  - Сила озарения (1-3 звезды)
- **5.5.3** User видит список озарений с:
  - Звёздами силы (1-3)
  - Превью текста
  - Иконкой камеры
- **5.5.4** User кликает озарение → bottom sheet с опциями:
  - Редактировать (текст и силу)
  - Удалить
- **5.5.5** User кликает иконку камеры → переход на экран камеры для фото озарения

#### 5.6 Вопросы (Questions)
- **5.6.1** User видит секцию "Вопросы" с кнопкой +
- **5.6.2** User кликает + → bottom sheet для добавления вопроса
- **5.6.3** User видит список вопросов с:
  - Иконкой статуса (✓ отвечен / ? без ответа)
  - Превью вопроса
  - Превью ответа (если есть)
- **5.6.4** User кликает вопрос → bottom sheet с:
  - Полным текстом вопроса
  - Ответом (если есть)
  - Кнопкой "Получить ответ" (выбор AI-персоны)
  - Кнопкой редактирования
  - Кнопкой удаления
- **5.6.5** User свайпает влево → закрытие карточки вопроса

#### 5.7 Подсказки (Hints)
- **5.7.1** User видит секцию "Подсказки" с кнопкой +
- **5.7.2** User кликает + → bottom sheet для запроса подсказки:
  - Текст заметки (что непонятно)
  - Выбор AI-персоны
- **5.7.3** User видит список подсказок с:
  - Иконкой статуса (✓ сгенерирована / ⚠ ожидает)
  - Превью заметки
  - Превью подсказки (если есть)
- **5.7.4** User кликает подсказку → bottom sheet с:
  - Полным текстом подсказки
  - Кнопкой "Пересоздать" (новая AI-персона)
  - Кнопкой редактирования заметки
  - Кнопкой удаления

#### 5.8 Фото решения
- **5.8.1** При отсутствии фото — User видит ListTile "Фото решения"
- **5.8.2** User кликает → переход на экран камеры
- **5.8.3** При наличии фото — User видит превью с кнопкой "Обновить"

#### 5.9 Завершение сессии
- **5.9.1** User кликает "Завершить сессию" → bottom sheet с оценками:
  - "Понравилось решение?" (👍/👎)
  - "Было ли озарение?" (1-3 звезды или 0)
  - "Насколько задача понравилась?" (1-5 звёзд)
  - "Насколько сложной была?" (1-5 звёзд)
- **5.9.2** User кликает "Завершить" → сохранение и переход на страницу решения
