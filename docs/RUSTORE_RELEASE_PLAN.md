### Скриншоты (минимум 2)

**Требования RuStore:**
- Соотношение сторон: **9:16** (портрет)
- Рекомендуемые размеры: 720×1280, 1080×1920, 1440×2560
- Формат: JPEG или PNG
- Минимум 2 скриншота, рекомендуется 4-5

Рекомендуемые экраны для скриншотов:
1. **Главный экран** — приветствие + активные задачи
2. **Библиотека** — список задач с карточками
3. **Сессия решения** — таймер + условие
4. **Профиль** — статистика + баланс
5. **Карта концептов** — граф знаний

#### Настройка размера экрана для скриншотов

**Вариант 1: Через ADB (рекомендуется)**

```bash
# Подключить устройство и проверить
adb devices

# Установить разрешение 1080x1920 (портрет, 9:16)
adb shell wm size 1080x1920

# Установить плотность пикселей (опционально, для чёткости)
adb shell wm density 420

# Сбросить на оригинальное разрешение после скриншотов
adb shell wm size reset
adb shell wm density reset
```

**Вариант 2: Через Flutter**

```bash
# Запустить приложение с фиксированным размером (для эмулятора)
flutter run -d <device_id> --device-vmservice-port=8888

# Или использовать эмулятор с нужным разрешением
flutter emulators --launch <emulator_id>
```

**Вариант 3: Через настройки разработчика на устройстве**

1. Включить "Режим разработчика" (Настройки → О телефоне → 7 раз нажать "Номер сборки")
2. Настройки → Для разработчиков → "Минимальная ширина" (sw)
3. Или: Настройки → Для разработчиков → "Разрешение экрана" (если есть)

**Для устройств Samsung:**
1. Настройки → Для разработчиков → "Разрешение экрана"
2. Или использовать Good Lock → MultiStar

**Для устройств Xiaomi:**
1. Настройки → Расширенные настройки → Для разработчиков
2. "Минимальная ширина" — установить значение 360-411

#### Создание скриншотов

```bash
# После настройки разрешения, сделать скриншот через ADB
adb shell screencap -p /sdcard/screen1.png
adb pull /sdcard/screen1.png screens/main_scr.png

# Или стандартной комбинацией кнопок (Power + Volume Down)

# Проверить размер скриншота
file screens/main_scr.png
identify screens/main_scr.png
```

#### Проверка соотношения сторон

```bash
# Скрипт проверки всех скриншотов в папке screens/
for f in screens/*.jpg screens/*.png; do
  if [ -f "$f" ]; then
    w=$(identify -format "%w" "$f")
    h=$(identify -format "%h" "$f")
    ratio=$(echo "scale=4; $w/$h" | bc)
    target="0.5625"
    echo "$f: ${w}x${h} ratio=$ratio (target: 0.5625)"
  fi
done
```

**Папка для скриншотов:** `screens/`

## 4. Процесс публикации

### Шаг 1: Подготовка APK
```bash
# Настроить подпись в android/app/build.gradle.kts
# Создать keystore
keytool -genkey -v -keystore lezhandr-release.jks -keyalg RSA -keysize 2048 -validity 10000 -alias lezhandr

# Собрать release APK
flutter build apk --release
```

### Шаг 2: Регистрация в RuStore
1. Создать аккаунт разработчика на https://developer.rustore.ru
2. Заполнить профиль разработчика
3. Добавить новое приложение

### Шаг 3: Загрузка ресурсов
1. Загрузить иконки
2. Загрузить скриншоты
3. Заполнить описания
4. Указать категорию и рейтинг

### Шаг 4: Загрузка APK
1. Загрузить `app-release.apk`
2. Заполнить версию и изменения
3. Отправить на модерацию

### Шаг 5: Модерация
- Обычно 1-3 рабочих дня
- Возможны запросы на доработку
