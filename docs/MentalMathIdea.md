# Обсуждение концепции: Mental Math (рабочее название)

## 1. Суть идеи

**Проблема:** Сложные математические задачи (например, упрощение выражений из Сканави) невозможно решить в уме, потому что требуют одновременно держать в голове множество промежуточных результатов и операций.

**Решение:** Декомпозиция сложной задачи на **граф навыков**, каждый из которых тренируется отдельно до автоматизма, а затем собирается обратно в целостное умение.

---

## 2. Архитектура навыков

### Пример: Упрощение выражения
```
Исходная задача: √(12 + 2√11) + √(12 - 2√11)

Декомпозиция на навыки:
├── Базовые арифметические навыки
│   ├── Сложение двузначных чисел в уме
│   ├── Умножение двузначных чисел в уме
│   └── Извлечение квадратов до 20²
├── Навыки работы с корнями
│   ├── Оценка √n (приближённо)
│   ├── Формула √(a ± 2√b) = √m ± √k
│   └── Подбор m и k таких, что m+k=a, mk=b
├── Рабочая память
│   ├── Удержание 2-3 промежуточных значений
│   ├── Параллельное вычисление двух ветвей
│   └── Сложение иррациональных выражений
└── Метанасыки
    ├── Выбор стратегии декомпозиции
    ├── Проверка результата
    └── Оценка "подъёмности" задачи
```

### Связь навыков
Навыки образуют **ориентированный ациклический граф (DAG)**:
- Некоторые навыки — prerequisites для других
- Несколько навыков могут комбинироваться в более сложные
- Путь от базовых к целевому навыку — не единственный

## 3. Игровая механика (Duolingo-стиль)

### Уровни навыка
```
Уровень 1: "Знакомство" — задачи с подсказками, медленный темп
Уровень 2: "Понимание" — меньше подсказок, умеренный темп
Уровень 3: "Уверенность" — минимум подсказок, нормальный темп
Уровень 4: "Скорость" — без подсказок, ограничение по времени
Уровень 5: "Мастерство" — автоматизм, высокая скорость
Уровень 6: "Виртуоз" — исключительная скорость и точность
```

### Прогрессия сложности
```python
# Пример: навык "Сложение двузначных чисел"

level_1 = [
    "23 + 15",  # без перехода через десяток
    "34 + 12",
    "41 + 25",
]

level_2 = [
    "27 + 18",  # с переходом через десяток
    "39 + 25",
    "48 + 34",
]

level_3 = [
    "67 + 89",  # большие числа
    "78 + 76",
    "95 + 87",
]

level_4 = [
    # три числа подряд
    "34 + 27 + 18",
    "45 + 38 + 29",
]

level_5 = [
    # скорость:限时 5 секунд
    "67 + 89",  # → 156
]
```

### Награды и мотивация
- **XP за каждый навык** → общий уровень
- **Streak** — ежедневная практика
- **Комбо** — несколько правильных ответов подряд
- **"Золотые навыки"** — навыки на уровне Мастера
- **"Достижения"** — особые milestones

## 4. Техническая архитектура

### Вариант A: Полноценный сервер (как MindVector)

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│   Flutter   │──────│   FastAPI   │──────│  PostgreSQL │
│   Client    │      │   Server    │      │  Database   │
└─────────────┘      └─────────────┘      └─────────────┘
                            │
                     ┌──────┴──────┐
                     │   AI/LLM    │
                     │  Generator  │
                     └─────────────┘
```

**Плюсы:**
- Синхронизация прогресса между устройствами
- Динамическая генерация задач через LLM
- Аналитика и адаптация под пользователя
- Социальные функции (рейтинги, соревнования)

**Минусы:**
- Сложность разработки
- Затраты на сервер и LLM
- Зависимость от интернета

---

### Вариант B: Минимальный сервер + AI на устройстве

```
┌─────────────────────────────────────┐
│            Flutter Client           │
├─────────────┬─────────────┬─────────┤
│   Local     │   Local     │  Sync   │
│   SQLite    │   LLM       │  API    │
│   Database  │  (Gemma)    │ (min)   │
└─────────────┴─────────────┴─────────┘
                      │
              ┌───────┴───────┐
              │  Minimal API  │
              │  (auth+sync)  │
              └───────────────┘
```

**Плюсы:**
- Работает офлайн
- Нет затрат на LLM-запросы
- Быстрый отклик
- Приватность данных

**Минусы:**
- Модель на устройстве (~2-4 GB)
- Ограниченные возможности генерации
- Сложнее обновлять контент

---

### Вариант C: Гибридный подход ⭐ **рекомендую**

```
┌────────────────────────────────────────────────────────┐
│                   Flutter Client                       │
├──────────────┬──────────────┬─────────────┬───────────┤
│   Local      │   Cached     │   Local     │  Offline  │
│   SQLite     │   Tasks      │   Inference │  Mode     │
│   (progress) │   Library    │   (simple)  │           │
└──────────────┴──────────────┴─────────────┴───────────┘
                      │
              ┌───────┴───────────────────────┐
              │         FastAPI Server        │
              │  ┌─────────────────────────┐  │
              │  │  Task Library (curated) │  │
              │  │  + AI Generator (LLM)   │  │
              │  │  + Progress Sync        │  │
              │  └─────────────────────────┘  │
              └───────────────────────────────┘
```

**Как это работает:**

1. **Библиотека задач** — кураторская, заранее созданная для каждого навыка
2. **AI-генерация** — только для создания новых вариаций на сервере
3. **Кэширование** — задачи скачиваются на устройство
4. **Офлайн-режим** — весь прогрессивный контент доступен без сети
5. **Синхронизация** — только прогресс и настройки

**Затраты:**
- Сервер: минимальный (тот же что MindVector)
- LLM: только для генерации новых задач (можно батчить)
- Хранилище: ~50-100 MB библиотека задач

## 6. Что нужно для MVP

### Минимальный набор навыков (MVP)

```
Арифметика в уме:
├── Сложение двузначных (6 уровней)
├── Вычитание двузначных (6 уровней)
├── Умножение двузначных (6 уровней)
└── Деление с остатком (4 уровня)

Квадраты и корни:
├── Квадраты до 20² (4 уровня)
├── Квадраты чисел, оканчивающихся на 5 (2 уровня)
└── Приближённые корни (3 уровня)

Всего: ~50 уровней × ~20 задач = ~1000 задач
```

### Технический стек MVP

```
Frontend: Flutter (переиспользование из Лежандра)
Backend: FastAPI (расширение MindVector или отдельный сервис)
Database: PostgreSQL (та же база, новые таблицы)
AI: GPT-4o-mini для генерации задач (дёшево, качественно)
```

### Оценка времени

| Этап | Время |
|------|-------|
| Проектирование графов навыков | 1-2 недели |
| Создание библиотеки задач (1000 шт) | 2-3 недели |
| Flutter-клиент (игровой интерфейс) | 3-4 недели |
| Backend API | 1-2 недели |
| Интеграция с Лежандром | 1 неделя |
| **Итого MVP** | **8-12 недель** |

## 7. Открытые вопросы

1. **Нужен ли AI в реальном времени?**
   - Если задачи кураторские — нет
   - Если генерируются динамически — да, но можно батчить

2. **Глубина декомпозиции?**
   - Насколько детально разбивать навыки?
   - Как избежать "переквалификации" (слишком мелкие навыки)?

3. **Адаптивность?**
   - Подстраиваться под слабые места пользователя?
   - Или линейная прогрессия для всех?

4. **Монетизация?**
   - Бесплатно: базовые навыки
   - Платно: продвинутые, AI-разбор своих задач

5. **Хостинг на сервере MindVector?**
   - Да, можно: та же PostgreSQL, общий auth
   - Новые таблицы: `mental_skills`, `mental_tasks`, `mental_progress`

## 8. Моя рекомендация

**Для старта: Вариант C (Гибридный)**

1. **Хостинг:** Тот же сервер MindVector
2. **База:** Та же PostgreSQL, новые таблицы
3. **Аутентификация:** Переиспользовать account_key
4. **Генерация задач:** LLM на сервере, кэширование на клиенте
5. **Офлайн:** SQLite на устройстве для прогресса и скачанных задач
6. **Клиент:** Flutter, можно взять основу из Лежандра

**Почему так:**
- Минимальные затраты на инфраструктуру
- Переиспользование кода и дизайна
- Постепенное усложнение (MVP → полноценный продукт)
- Возможность интеграции с Лежандром в будущем
